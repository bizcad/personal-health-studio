# Contributing

This is a personal project for health data intelligence. Contributions are welcome in the form of:

- Bug reports (via Issues)
- Documentation improvements
- Code quality enhancements
- Security recommendations

## Standards

All contributions should follow:

1. **Principles-and-Processes.md** - Engineering standards
2. **CLAUDE.md** - Design philosophy
3. **Python type hints** - Full type annotations required
4. **Docstrings** - Every function must be documented
5. **Tests** - Extraction accuracy is non-negotiable

## Process

1. Fork the repository
2. Create a feature branch
3. Make changes following the standards above
4. Add tests for critical functionality
5. Submit a pull request with clear description

## Health Data Handling

This codebase handles sensitive personal health information. All contributions must:

- Preserve data accuracy (no approximations)
- Maintain privacy (no external data sharing)
- Log security-relevant operations
- Never expose credentials in code or logs
- Include security review checklist

See README.md for overview or CLAUDE.md for design context.
